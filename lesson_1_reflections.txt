Why do you think some version control systems, like Git, allow saving multiple
files in one commit, while others, like Google Docs, treat each file separately?

    A program will be most likely composed by various files, and sometimes the changes created will spread through more than one file. Git will be used to track software development projects, whereas Google Docs will be used for Documents

How can you use the commands git log and git diff to view the history of files?

    Log shows the list of commited files, whereas diff shows the concrete file difference.
